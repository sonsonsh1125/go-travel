# Go-Travel 프로젝트 컨텍스트

## 프로젝트 개요

YouTube 영상 설명에서 AI를 사용하여 여행지 장소를 자동 추출하는 Next.js 웹 애플리케이션

## 핵심 아키텍처

### 데이터 흐름
1. 사용자 입력 (예: "교토")
2. YouTube Data API로 영상 검색 (10개, 조회수 상위)
3. 영상 설명(description) 수집
4. OpenAI GPT-4o-mini로 장소 추출
5. 중복 제거 및 언급 횟수 집계
6. 웹 UI에 결과 표시 + 구글 지도 링크

### 기술 제약사항

**Vercel 서버리스 환경 최적화**:
- ❌ youtube-transcript 라이브러리 사용 불가 (웹 스크래핑 제한)
- ✅ YouTube Data API의 영상 설명(description) 사용
- ⏱️ 무료 플랜 10초 타임아웃 제한 → 영상 10개로 제한
- ⚡ AI 요청 간 300ms 딜레이로 속도 최적화

## AI 장소 추출 필터링 규칙

`PlaceExtractorService`의 `buildPrompt` 메서드에서 OpenAI에 전달하는 엄격한 규칙:

### ✅ 반드시 포함되어야 하는 장소

1. **구체적인 상호명이 있는 장소만**
   - 예: "이치란 라멘 교토점", "니시키시장", "기요미즈데라"
   - 예: "도쿠라 교토 산조점", "쿄센도", "스시 이시마츠"

2. **실제 방문 가능한 장소**
   - 주소가 있고 방문할 수 있는 구체적인 상호

3. **검색 지역 내부 또는 근교만**
   - 검색한 도시 시내 또는 근교(30분 이내)
   - 예: "교토" 검색 시 교토 시내 및 근교만

### ❌ 반드시 제외되어야 하는 장소

1. **검색어 자체만 있는 지역명**
   - ❌ 예: "교토" 검색 시 "교토"만 있는 경우
   - ✅ 예: "교토" 검색 시 "아라시야마", "기온", "후시미" 등 하위 지역명은 허용
   - ❌ 예: "오사카" 검색 시 "오사카"만 있는 경우
   - ✅ 예: "오사카" 검색 시 "도톤보리", "신사이바시" 등 하위 지역명은 허용

2. **검색 지역 외부 장소**
   - 예: "교토" 검색 시 오사카, 나라, 도쿄의 장소는 제외

3. **호텔/숙박 시설 (매우 중요!)**
   - 예: "료칸", "호텔", "게스트하우스", "민박"
   - 카테고리가 "호텔"인 장소는 절대 포함하지 말 것

4. **애매한 표현**
   - 예: "그 카페", "유명한 곳", "거기", "그곳"
   - 예: "편의점", "백화점" (구체적인 상호명 없음)

### 🎯 추출 제한 및 균형

1. **영상당 최대 3개 장소**
   - 한 영상에서 너무 많은 장소를 추출하지 않음
   - 가장 중요하고 구체적인 장소만 선별

2. **카테고리 균형 유지**
   - 맛집만 3개보다는 다양한 카테고리 혼합
   - 예: 맛집 1개 + 관광지 1개 + 사찰/액티비티 1개

3. **우선순위**
   - 1순위: 관광지, 사찰
   - 2순위: 맛집, 카페
   - 3순위: 쇼핑, 기타

### 카테고리 분류

추출된 장소는 반드시 다음 카테고리 중 하나로 분류:

- **맛집**: 레스토랑, 라멘집, 식당 등
- **카페**: 커피숍, 디저트 카페
- **관광지**: 유명 관광 명소, 전망대, 공원
- **사찰**: 신사, 절, 종교 시설
- **시장**: 전통 시장, 재래시장
- **쇼핑**: 쇼핑몰, 백화점, 상점가 (구체적인 상호명 있는 경우만)
- **액티비티**: 체험 시설, 오락실
- **기타**: 역, 공원, 기타 장소

## 주요 파일 구조

```
web/
├── app/api/search/route.ts     # 메인 API 엔드포인트
├── lib/services/
│   ├── youtube.ts              # YouTube 검색
│   ├── transcript.ts           # 영상 설명 수집
│   └── placeExtractor.ts       # AI 장소 추출 (필터링 규칙 포함)
└── components/
    ├── SearchForm.tsx          # 검색 폼
    └── Results.tsx             # 결과 표시 + 지도 링크
```

## 환경 변수

- `YOUTUBE_API_KEY`: YouTube Data API v3 키
- `OPENAI_API_KEY`: OpenAI API 키
- `AI_MODEL`: 사용할 AI 모델 (기본값: gpt-4o-mini)

## 개발 시 주의사항

1. **필터링 규칙 변경 시**: `web/lib/services/placeExtractor.ts`의 `buildPrompt` 메서드 수정
2. **영상 설명 길이**: 처음 8000자만 AI에 전달 (토큰 비용 최적화)
3. **영상 설명 최소 길이**: 50자 미만인 경우 자동 스킵
4. **Vercel 배포**: 환경 변수 설정, Root Directory를 `web`으로 설정
5. **API 비용**: 영상 10개 처리 시 약 $0.02-0.05 (설명 길이에 따라 다름)
