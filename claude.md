# Go-Travel 프로젝트 요구사항 명세서

## 프로젝트 개요

YouTube에서 특정 여행지 관련 영상을 검색하고, 각 영상의 자막에서 언급된 장소들을 자동으로 추출하여 Notion 데이터베이스에 정리하는 자동화 시스템

## 핵심 기능

### 1. 여행지 입력
- 사용자가 원하는 여행지를 입력 (예: "교토", "오사카", "파리", "방콕")
- CLI 인터페이스를 통한 대화형 입력 또는 명령행 인자로 전달

### 2. YouTube 영상 검색
- 입력된 여행지를 기반으로 YouTube 검색
  - 검색 쿼리 예시: "{여행지} 여행", "{여행지} 브이로그", "{여행지} 맛집"
- 조회수 기준 상위 30개 영상 수집
- 수집할 정보:
  - 영상 ID
  - 제목
  - 채널명
  - 조회수
  - 게시일
  - 썸네일 URL
  - YouTube URL

### 3. 자막(Transcript) 수집
- 각 영상의 자막 추출
  - 우선순위: 한국어 → 영어 → 기타
- 자막이 없는 영상은 스킵
- 전체 자막 텍스트 수집

### 4. 장소 추출 (핵심 기능)
- AI를 활용하여 자막에서 언급된 장소들만 추출
- 추출할 정보:
  - **장소명** (예: "기요미즈데라", "니시키시장", "아라시야마")
  - **장소 카테고리** (예: "사찰", "시장", "관광지", "맛집", "카페", "호텔")
  - **언급된 영상** (어떤 영상에서 언급되었는지)
  - **언급 횟수** (여러 영상에서 언급되면 카운트)
  - **간단한 설명** (자막에서 추출한 해당 장소에 대한 설명)

### 5. Notion 데이터베이스 저장
- 추출된 장소 정보를 Notion Database에 저장
- Database 구조:
  - **장소명** (Title) - 장소 이름
  - **카테고리** (Select) - 사찰/시장/맛집/카페/관광지/호텔/기타
  - **여행지** (Select) - 검색한 여행지 (교토/오사카/파리 등)
  - **언급 횟수** (Number) - 몇 개의 영상에서 언급되었는지
  - **언급 영상** (Multi-select 또는 Relation) - 어떤 영상들에서 언급되었는지
  - **설명** (Text) - 장소에 대한 간단한 설명
  - **추가일** (Date) - 데이터 추가 날짜

## 기술 스택

### 필수
- **Node.js/TypeScript** - 기본 런타임 및 언어
- **YouTube Data API v3** - 영상 검색 및 정보 수집
- **youtube-transcript** - 자막 추출
- **Notion API (@notionhq/client)** - Notion 데이터베이스 연동
- **OpenAI API 또는 Anthropic Claude API** - 자막에서 장소 추출

### 선택
- **dotenv** - 환경 변수 관리
- **axios** - HTTP 요청

## 환경 변수

```env
# YouTube Data API v3
YOUTUBE_API_KEY=your_youtube_api_key

# Notion Integration
NOTION_API_KEY=your_notion_integration_token
NOTION_DATABASE_ID=your_notion_database_id

# AI API (OpenAI 또는 Anthropic 중 선택)
OPENAI_API_KEY=your_openai_api_key
# 또는
ANTHROPIC_API_KEY=your_anthropic_api_key

# 검색 설정
MAX_RESULTS=30
```

```
[
  {
    "name": "장소명",

- 구체적인 장소명만 추출 (애매한 표현은 제외)

- 중복 제거
```

## 데이터 흐름

```
1. 사용자 입력 (여행지)
   ↓
2. YouTube 검색 (여행지 + "여행")
   ↓
3. 상위 30개 영상 정보 수집
   ↓
4. 각 영상의 자막 수집
   ↓
5. AI API로 각 자막에서 장소 추출
   ↓
6. 장소 데이터 정규화 및 중복 제거
   ↓
7. Notion Database에 저장
   ↓
8. 완료 리포트 출력
```

## 추가 개선 사항 (선택)

### Phase 2
- 지도 API 연동으로 장소 좌표 수집
- 중복 장소 자동 병합 (예: "기요미즈데라", "청수사" → 동일 장소)
- 장소별 추천 시간대 추출
- 예산 정보 추출

### Phase 3
- 웹 UI 제공
- 여러 여행지 동시 검색
- 장소별 상세 페이지 자동 생성
- 여행 루트 최적화 제안

## 제약사항

1. **YouTube API 할당량**: 일일 10,000 units (영상 30개 검색 시 약 200 units 소모)
2. **AI API 비용**: 자막 길이에 따라 토큰 비용 발생
3. **Notion API Rate Limit**: 초당 3회 요청 제한
4. **자막 제공 여부**: 모든 영상에 자막이 있는 것은 아님

## 성공 기준

- [ ] 사용자가 여행지를 입력하면 자동으로 검색
- [ ] 30개 영상의 자막 수집 (최소 20개 이상)
- [ ] AI를 통해 장소 정확하게 추출 (정확도 90% 이상)
- [ ] Notion에 중복 없이 깔끔하게 정리
- [ ] 전체 프로세스 10분 이내 완료

## 참고 사항

- 첫 실행 시 Notion Database를 수동으로 생성하거나, 코드에서 자동 생성 기능 구현 필요
- AI API 선택 시 비용 고려 (OpenAI GPT-3.5 or GPT-4, Anthropic Claude)
- 장소 추출 정확도를 높이기 위해 프롬프트 엔지니어링 중요
